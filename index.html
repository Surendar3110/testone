Your quick reference for resolving IT issues.

Issue: Contact Pad Status Not Updating in Edge Browser

Problem: Status was not visually updating in Vonage Contact Pad.

Solution: Update Edge browser to latest version, then log out and log back in.

Result: Status updates correctly after these steps.

Issue: Call Routing Failure in Vonage

Problem: Calls not routed due to device busy status caused by headphone conflicts.

Solution: Check user system settings related to device access and headphone use. Schedule session to update settings.

Issue:Calls Are Landing on Mobile Instead of Vonage Contact Pad

Problem: Incoming calls are routed to the user’s mobile phone instead of being handled through the Vonage Contact Pad in Salesforce

Solution: Check "Use Softphone" Setting

1. Open the Vonage Contact Pad in Salesforce.
2. Locate the "Use Softphone" option.
3. Ensure the checkbox is ticked (enabled).

o ✅ If "Use Softphone" is enabled: Calls will now route through the Vonage CP.
o ❌ If not enabled: Enable it to ensure calls are received via the contact pad.

Result: Status updates correctly after these steps.

Additional Notes:
• Users must have proper browser permissions and pop-up settings enabled for Vonage CP to work correctly.
• Salesforce and Vonage may occasionally require re-authentication after updates or cache clears.

Issue: Analyze Agent Logged-in Status and Call CDR Reports

Problem: Need to analyze the agent’s logged-in status and call CDR (Call Detail Records) reports.

Solution:
1. Log in to the Vonage Portal.

2. Navigate to the following dashboards to verify agent status and call data:
o Call Analysis with Call GUID
o Agent Performance – Agent Timelines
o Delivery failures summary

3. Use these dashboards to review agent activity, call details, and performance metrics.

Result: Status updates correctly after these steps.
